## 1. 一句话判断

可以冲，但要按“高强度长期项目”执行，而不是按普通期末复习思路执行。

你的优势：

- 大二下开始，仍有完整的第一轮、强化、真题、考前收束时间。
- 信管背景与软件工程、数据系统、信息系统类项目有一定连接点。
- 目标明确，能提前建立数学、408 和复试项目三条线。

你的劣势：

- 跨考，408 四门需要从零搭体系。
- 数学一要求高，420 目标下数学不能偏弱。
- 北大软微竞争强，复试也不能空着。

核心结论：

- **420 的底层逻辑不是“每科都不错”，而是数学一和 408 足够强。**
- 推荐分数结构：政治 75、英语一 80、数学一 130、408 135。
- 如果 2027 年 8 月全科模拟还不到 390，需要准备稳妥院校梯队。
- 如果 2027 年 10 月全科模拟还不到 405，不建议只押北大软微。

## 4. 总学习量预算

从 2026 年 6 月 8 日到 2027 年 12 月，建议按 **2660 小时有效学习** 设计。
如果执行力很强，冲到 2800 小时更稳；如果低于 2200 小时，420 风险会明显升高。

有效学习包括：

- 做题
- 复盘
- 默写
- 限时训练
- 错题回炉
- 项目编码和复试表达训练

不计入有效学习：

- 长时间浏览备考帖
- 反复比较老师
- 只开视频不动笔
- 整理无法复用的笔记
- 长时间整理资料目录

## 5. 资料清单：少而准

1. 每科只选一个主体系。
2. 不提前囤满所有考前资料。

4. 视频只是入口，题目和复盘才是学习。
5. 一份资料完成 70% 以上，再考虑加资料。

### 5.5 编程与复试项目资料

你需要一个能在复试中讲清楚的计算机项目。项目不追求复杂，但要真实、能运行、能解释。

基础能力：

- C 语言：数组、指针、结构体、递归。
- 数据结构代码：顺序表、链表、栈、队列、二叉树、图、排序。
- Git：提交、分支、README。
- 数据库：MySQL 或 PostgreSQL，表设计、索引、事务。
- Web 基础：HTTP、接口、前后端交互。

项目建议：

| 项目 | 难度 | 复试价值 |
|---|---|---|
| 课程管理系统 | 中 | 贴合信管背景，可讲数据库和业务建模 |
| 图书管理系统 | 低到中 | 适合展示 CRUD、权限、表设计 |
| 算法可视化工具 | 中 | 展示数据结构理解 |
| 简易论坛或问答系统 | 中 | 可讲接口、数据库、登录权限 |
| 备考进度监测网页 | 中 | 可以和本计划结合，后续直接使用 |

项目最低标准：

- 有 Git 仓库。
- 有 README。
- 能本地运行。
- 有数据库或核心算法。
- 有 3 个可讲技术点。
- 有 3 个可讲改进方向。

## 8. 快速总计监测方法

### 8.1 每日记录表

字段说明：

### 8.2 Excel 公式

假设：

- A 列：日期
- C 列：数学min
- D 列：408min
- E 列：英语min
- F 列：政治min
- G 列：项目min
- H 列：总min
- I 列：核心min
- M 列：新增错题
- N 列：回炉错题

每日总时长：

```text
=SUM(C2:G2)
```

每日核心时长：

```text
=C2+D2
```

每日核心占比：

```text
=IF(H2=0,0,I2/H2)
```

最近 7 天总小时：

```text
=SUMIFS($H:$H,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY())/60
```

最近 7 天数学+408 小时：

```text
=SUMIFS($I:$I,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY())/60
```

最近 7 天核心占比：

```text
=IF(SUMIFS($H:$H,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY())=0,0,SUMIFS($I:$I,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY())/SUMIFS($H:$H,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY()))
```

最近 7 天错题回炉率：

```text
=IF(SUMIFS($M:$M,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY())=0,0,SUMIFS($N:$N,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY())/SUMIFS($M:$M,$A:$A,">="&TODAY()-6,$A:$A,"<="&TODAY()))
```

本月总小时：

```text
=SUMIFS($H:$H,$A:$A,">="&EOMONTH(TODAY(),-1)+1,$A:$A,"<="&EOMONTH(TODAY(),0))/60
```

累计总小时：

```text
=SUM($H:$H)/60
```

距离初试天数：如果 B1 填一个推算排程日，例如 `2027-12-25`，必须标注“非官方初试日期”：

```text
=$B$1-TODAY()
```

如果你的 Excel 或 WPS 用分号分隔参数，把公式里的逗号替换成分号。

- 树和二叉树
- 图
- 查找
- 排序
- 算法题表达

- 数据表示和运算
- 存储系统
- 指令系统
- CPU

操作系统：

- 进程和线程
- 处理机调度
- 同步互斥
- 死锁
- 内存管理
- 虚拟内存
- 文件系统
- I/O 管理

计算机网络：

- 物理层基础

- 路由协议

## 15. 后续网页规格

后续可以把这个 Markdown 做成备考驾驶舱。

首页显示：

- 本周有效小时。
- 数学+408 占比。
- 本月目标完成率。

核心页面：

1. 总览页：目标分、当前均分、累计小时、风险状态。
2. 每日记录页：输入各科分钟数、题量、错题数。
3. 周复盘页：自动统计周总小时、核心占比、错题回炉率。
4. 月度路线页：展示 2026-06 到 2027-12 的里程碑。
5. 资料管理页：记录每本资料完成率。
6. 模拟成绩页：记录各科分数，自动算近 5 套和近 10 套均分。
7. 风险预警页：根据时间、小时、分数给出绿黄红状态。
8. 复试材料页：项目、简历、自我介绍、问题库。

网页定位：

- 不是宣传页。
- 不是好看的空壳。
- 是每天能打开填数据、看偏航、调整任务的学习推进工具。

### 15.1 网站升级后的使用原则

网站必须服务执行，而不是制造新的整理负担。所有页面只围绕四个问题：

1. 今天该学什么。
2. 今天是否记录。
3. 本周是否偏航。
4. 学过的内容是否按记忆曲线回炉。

因此网页的核心信息架构固定为：

| 区域 | 用途 | 每天是否使用 |
|---|---|---|
| 总览 | 看阶段、风险、本周小时、核心占比、考纲完成度 | 每天 30 秒 |
| 今日任务 | 执行任务、保存分钟数、题量、错题数 | 每天 |
| 零基路线 | 判断当前能力层级，不跨层幻想 | 每周 |
| 考纲与能力图谱 | 点击标记未开始、需复盘、已掌握 | 每 2-3 天 |
| 记录库 | 查看历史记录、导出备份、修正错误记录 | 每周 |
| 复盘 | 看周指标、月度里程碑、记忆曲线队列 | 每周日 |
| 模考 | 2027 年强化期后记录套卷分数 | 有套卷时 |
| 资料 | 跟踪资料完成率，避免资料过载 | 每周 |
| 设置 | 自定义可用时间、任务数、复盘间隔 | 节奏变化时 |

### 15.2 每日任务生成规则

网页生成任务时必须遵守：

- 每日必做任务默认 3 项，可设置为 3-4 项；顺延或底线日可以降到 2 项。
- 系统按日期渐进加量：2026 年 6 月 8 日后工作日约 90 分钟、周末约 150 分钟；7 月约 120/210 分钟；8 月约 150/240 分钟；9 月起工作日约 270 分钟、周末约 390 分钟。
- 数学和 408 的总占比默认不低于 65%。
- 任务必须包含“学习、练习、验收”三段，不允许只有“看视频”。
- 零基础启动期优先推：函数和极限预备、C 语言变量/循环/数组、顺序表概念、英语单词和长难句。
- 如果本周某个核心科目低于配额 70%，自动加入补弱任务。
- 如果存在到期复盘，今日任务优先加入 1 项复盘。

最低任务量：

| 日期类型 | 最低可执行任务 |
|---|---|
| 状态差 | 数学 45m、408 45m、英语 20m |
| 2026 年 6 月 8 日后工作日 | 数学 45m、408 45m、英语 20m |
| 顺延日 | 先做顺延任务 + 1 个核心任务，必要时只保 2 项 |
| 9 月后工作日 | 数学 110-140m、408 110-140m、英语 30-50m、复盘 10-20m |

默认 3 项不是降低标准，而是把任务从“看起来完整”改成“当天一定能闭环”：

1. 数学主任务。
2. 408 主任务。
3. 英语或到期复盘。

只有在当天可用时间充足、且前 3 项已经稳定完成时，才把第 4 项作为政治、复盘、补弱或项目任务。

### 15.3 记忆曲线复盘规则

网页中的任务一旦勾选完成，会自动进入复盘队列：

| 轮次 | 目的 | 建议时长 |
|---|---|---:|
| D+1 | 防止刚学完就遗忘，重做最小错题集 | 5-15m |
| D+3 | 检查是否还能识别题型和概念触发点 | 10-20m |
| D+7 | 周复盘合并同类错因 | 15-25m |
| D+14 | 从题目回到章节框架，补知识图 | 15-25m |
| D+30 | 月度回炉，只保留高频错题和核心公式 | 20-35m |

复盘原则：

- 复盘不是重看视频。
- 数学复盘重做错题、默写公式、写识别信号。
- 408 复盘画过程图、写伪代码、解释代价。
- 英语复盘重读定位句、长难句和错选项。
- 政治复盘以选择题错题和后期主观题关键词为主。

### 15.4 信息分层和前端设计原则

网页界面要保持浅色、克制、信息密度可控。它不是后台报表，也不是资料百科，而是一个轻量学习工作台：帮助你制定当天计划、监督执行、记录结果、触发复盘、分析偏航。

默认界面不是把所有资料一次性摊开，而是按“马上执行、420 内部校准线、快速监测、完整追溯”四层组织。

| 层级 | 默认是否展示 | 内容 | 目的 |
|---|---|---|---|
| 第一层：行动 | 展示 | 今日第一任务、1-2 个配套任务、本周状态 | 打开网页 10 秒内知道现在该做什么 |
| 第二层：校准 | 展示摘要 | 420+ 内部校准线、今日检查 | 判断当前风险是否仍可控 |
| 第三层：监测 | 展示核心，图表折叠 | 周时长、风险、核心占比 | 每天 1 分钟判断是否偏航 |
| 第四层：详情 | 默认折叠 | 数据闭环、进度环、科目分布、热力图、目标分、系统规则 | 周复盘或月复盘时再看 |
| 第五层：完整资产 | 按页查看 | 考纲与能力图谱、资料进度、记录库、模考库 | 需要追溯时打开，不干扰日常执行 |

默认折叠规则：

- 首页专注模式只直接显示下一步行动、420 内部校准线摘要、今日检查和 3 个核心指标。
- 目标分结构、距离初试、阶段建议在平衡或详尽模式查看。
- 数据闭环、图表、周配额、目标分、系统规则、热力图默认放入折叠区。
- 今日页只直接显示今日计划、执行顺序、学习输入和到期复盘。
- 今日建议细则、首月清单、记忆曲线说明、最近记录默认折叠。
- 考纲页只默认展开“当前下一步”或“有需复盘考点”的分组。

视觉原则：

- 整体保持浅色、安静、低边框、低阴影，不做旧式后台管理界面。
- 导航像工作台入口，不像表格菜单；移动端允许换行，不截断文字。
- 首页优先展示行动和风险，不用大面积指标堆叠制造压迫感。
- 今日页采用“左侧任务、右侧记录和复盘”的工作台布局。
- 字体层级只保留必要差异：页面标题、模块标题、指标数字、说明文字。
- 卡片只用于真实信息容器，避免为了装饰而堆卡片。
- 动画只用于视图进入、悬停、进度变化，轻微即可，不干扰学习。
- 任务和记录表单必须便于快速点击、快速输入、快速复盘。

这套分层参考成熟界面的渐进披露思想：先给用户当前需要的少量决策，再允许用户按需展开细节。考研系统不是百科页，默认状态必须服务“今天真的学完”。

### 15.4.1 10 秒日常使用流程

每天打开网页后只做这 4 步：

1. 看“下一步行动”，直接开始第一任务。
2. 完成后勾选任务，让系统自动生成 D+1/D+3/D+7/D+14/D+30 复盘。
3. 在“今日学习输入”填分钟、题量、新增错题、回炉错题。
4. 看“当前风险”：绿色继续，黄色补核心，红色只保数学和 408。

每天不需要做的事：

- 不需要重新研究长期计划。
- 不需要每天翻完整考纲。
- 不需要每天看所有图表。
- 不需要因为某一天没完成就补偿式加 8 个任务。

### 15.4.2 信息密度模式

网站必须支持三种密度，而不是只给一种固定界面：

| 模式 | 适用场景 | 默认显示 | 默认隐藏 |
|---|---|---|---|
| 专注 | 日常默认、开始学习、状态一般、时间紧 | 第一任务、420 内部校准线摘要、今日检查、核心指标、记录入口 | 目标分结构、趋势图、工作流、任务细则、考纲交付说明 |
| 平衡 | 周中检查或需要多看一点 | 今日计划、420 内部校准线、数据保险摘要、核心图谱 | 深层规则和完整细节 |
| 详尽 | 周复盘、月复盘、查考纲、调整计划 | 所有折叠区、任务学法、考纲解释和交付 | 不隐藏 |

密度切换原则：

- 专注模式不是减少目标，而是减少当下干扰。
- 专注模式用于 80% 的日常学习。
- 平衡模式用于周中检查或任务调整。
- 详尽模式只在复盘和查漏时使用，不建议每天长时间停留。
- 密度模式必须保存在本地，下次打开沿用上次选择。

### 15.4.3 数据保险和全流程闭环

网站不是一次性计划页，而是长期使用的数据系统。必须显示：

| 模块 | 记录内容 | 风险提示 |
|---|---|---|
| 学习记录 | 每日分钟、题量、错题、明日任务 | 连续 3 天不记录，视为节奏中断 |
| 考纲状态 | 未开始、需复盘、已掌握 | 只标“已掌握”但没题量，不算真正掌握 |
| 复盘队列 | D+1/D+3/D+7/D+14/D+30 | 到期复盘积压时，先复盘再开新内容 |
| 模考盘 | 分科分数和总分 | 2027 年 6 月后才作为主监测 |
| 数据备份 | JSON 导出日期 | 超过 7 天未导出，需要备份 |

全流程顺序固定为：

1. 计划：生成今日 3-4 项任务，默认只保 3 个必做任务。
2. 执行：在今日计划卡直接勾选完成。
3. 记录：保存分钟、题量、新错题、回炉错题。
4. 复盘：完成任务自动进入多轮复盘队列。
5. 模考：到阶段后记录套卷分数。
6. 校准：根据周指标、考纲进度、模考趋势调整下周。

### 15.5 图形化看板规则

网站必须把关键数据变成直观图形，减少“看一堆数字还不知道哪里偏航”的问题。

总览页必须包含：

| 图形 | 目的 | 绿色判断 |
|---|---|---|
| 总量进度环 | 看 2660h 总目标推进 | 不长期低于月度累计线 |
| 本月进度环 | 看当月目标完成率 | 月中约 50%，月底 90%+ |
| 本周进度环 | 看阶段周目标 | 周日达到 90%+ |
| 核心占比环 | 看数学+408 是否被挤压 | 65%+ |
| 本周科目分布图 | 看每科投入结构 | 数学和 408 合计最大 |
| 14 天趋势图 | 看连续性和递进 | 中断少，低谷可解释 |
| 热力图 | 看 28 天节奏 | 至少 6 天/周有记录 |

看图规则：

- 如果总量落后，但本周健康，继续执行，不临时加资料。
- 如果本周图红，但总量还没落后，下一周只补执行。
- 如果核心占比低于 55%，停止加英语、政治、项目任务，先补数学和 408。
- 如果趋势图连续 3 天为 0，先恢复底线日，不追求补偿式熬夜。

### 15.6 任务作业单规则

每日任务不只是“学某章”，必须变成作业单：

| 科目 | 必须显示 | 最低交付 |
|---|---|---|
| 数学 | 学什么、做什么、验收什么 | 基础题 15-25 道 + 错因 + 公式默写 |
| 408 | 学什么、画什么、写什么 | 章节题 20 道或 1 个伪代码/过程图 |
| 英语 | 背什么、读什么、复盘什么 | 单词 + 长难句/阅读定位句 + 错选项原因 |
| 政治 | 框架、选择题、关键词 | 选择题错题归类 |
| 复盘 | 回炉对象、错因、下一步 | 回炉 5-10 道错题 + 明日第一任务 |

任务卡显示规则：

| 区域 | 默认内容 | 作用 |
|---|---|---|
| 今日计划卡 | 科目、任务、分钟、最低交付、完成勾选 | 当天执行主入口 |
| 进度条 | 完成任务数、已完成分钟、完成率 | 快速判断今天是否收束 |
| 学法详情 | 学法、练习、验收 | 默认可折叠，详尽模式展开 |
| 任务结构图 | 今日任务顺序 | 用于防止并行切换和临时加量 |
| 任务细则 | 完整作业单和过关标准 | 周复盘或查漏时查看 |

任务执行顺序：

1. 先做数学或 408 中更难的一项。
2. 再做另一门核心科目。
3. 再做英语或政治。
4. 最后做复盘和明日第一任务。

不允许：

- 只写“看视频 2 小时”。
- 一天给自己排 8-10 个任务。
- 做完不记录题量和错题。
- 没完成旧任务就临时开新资料。
- 勾选完成但没有任何可交付产物。

### 15.7 考纲图谱规则

考纲与能力图谱需要显示：

- 官方范围映射、预备能力、备考补充三类标签。
- 高数预备、C 与算法预备标为预备能力；数学、408、英语、政治主干条目标为官方范围映射。
- 条目为备考拆解，不等同官方逐字大纲。

- 当前科目总完成率。
- 已掌握数量。
- 需复盘数量。
- 未开始数量。
- 每个分组完成率。
- 下一步 4 个小任务。
- 每个知识点的零基础解释。
- 每个知识点的最低交付。
- 每个知识点的建议训练量。

考点状态定义：

| 状态 | 含义 | 下一步 |
|---|---|---|
| 未开始 | 没学或只听过名字 | 进入每日任务推进 |
| 需复盘 | 学过但不稳、错题多 | 优先进入复盘 |
| 已掌握 | 能做题、能解释、能复现 | 后期套卷回炉 |

知识点卡规则：

| 密度 | 显示内容 |
|---|---|
| 专注 | 知识点名称、状态、训练量 |
| 平衡 | 知识点名称、零基础解释、状态、训练量 |
| 详尽 | 知识点名称、零基础解释、最低交付、状态、训练量 |

“已掌握”必须同时满足：

- 数学：能闭卷写公式或定理条件，基础题正确率稳定 80%+，错题能写识别信号。
- 408：能画结构图或过程图，能写伪代码/步骤，能说明复杂度或代价。
- 英语：能定位原文依据，解释错选项，不只是读懂大意。
- 政治：选择题错因能归类，后期主观题能写关键词。

零基础阶段不要追求考纲百分比好看。正确顺序是：

1. 先把数学预备、高数极限/导数、C 语言、线性表打通。
2. 再扩展到数据结构完整一轮和高数主干。
3. 再推进计组、OS、计网和数学三大模块闭合。
4. 最后进入真题、套卷、错题专题。

### 15.8 首屏与闭环看板规则

首屏必须回答三个问题：

| 问题 | 页面模块 | 判断方式 |
|---|---|---|
| 现在先学什么 | 下一步行动 | 每天只给 1 个第一任务和 2 个配套任务 |
| 是否仍在 420+ 轨道 | 420+ 内部校准线 | 看累计小时、阶段节点、目标分和核心占比 |
| 今天有没有形成有效记录 | 今日检查 | 任务、记录、复盘、核心占比四项检查 |

首屏不再直接堆满所有图表。数据闭环、趋势图、完整监测都可以展开，但默认不打断当天执行。

今日检查四项含义：

| 项目 | 可执行定义 |
|---|---|
| 任务 | 至少完成今日主任务和一个核心配套任务 |
| 记录 | 保存当天各科分钟、题量、错题和明日第一任务 |
| 复盘 | 到期复盘清空或明确顺延 |
| 核心 | 最近 7 天数学 + 408 占比达到设置目标，默认 65% |

如果今日检查低于 2/4，第二天自动按“底线日”执行：数学 45-60m、408 45-60m、英语 20m、复盘 10m，避免第二天任务超载。  
如果有未完成任务顺延，第二天先处理顺延任务，同时削减新增内容；必要时固定为 2 项，直到当天重新恢复稳定执行。

## 17. 网站部署与数据备份

当前网站已经升级为 **Vite + Supabase + Vercel**。不要再用普通静态服务直接打开项目根目录；那会让浏览器无法解析 `@supabase/supabase-js` 这类模块导入，页面会停在默认状态，路由和任务不会执行。

核心原则：前端用 Vite 构建，登录后数据写入 Supabase；未登录时仍可用 `localStorage` 作为本机草稿。

### 17.1 本机使用

在项目目录运行：

```powershell
npm install
npm run start:local
```

脚本会自动做四件事：

- 如果还没有 `node_modules`，先执行 `npm install`。
- 从 5173 开始寻找空闲端口。
- 用找到的端口启动 Vite。
- 等页面可访问后自动打开浏览器。

如果想从别的端口开始：

```powershell
powershell -ExecutionPolicy Bypass -File .\start-local.ps1 -StartPort 5200
```

以后调试页面必须使用这个脚本或 Vite 地址，不使用普通静态服务直接跑根目录。

生产检查：

```powershell
npm run build
npm run preview
```

### 17.2 Supabase 配置

1. 新建 Supabase 项目。
2. SQL Editor 执行 `supabase/schema.sql`。
3. Authentication 开启 Email Password。
4. 复制 Project URL 和 Publishable key。
5. 本地新建 `.env`：

```env
VITE_SUPABASE_URL=https://你的项目.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=你的 publishable key
```

只使用 publishable key，不把 service role key 放进前端。

### 17.3 Vercel 部署

远程仓库目标：

```text

```

Vercel 设置：

| 项目 | 设置 |

### 17.4 数据备份规则

云同步不是放弃备份。执行规则：

| 场景 | 操作 |
|---|---|
| 每周日晚上 | 点击“导出数据”，保存 JSON |

| 大量修改考纲状态前 | 先导出一份备份 |

强备份频率：每次月复盘后 1 次。

1. 今天先做什么。
2. 做到什么程度算完成。
3. 完成后什么时候复盘。

### 18.5 长期执行原则

去掉空泛文案的标准不是语气更像人，而是任务必须能被完成和验收。

| 不合格任务 | 合格任务 |
|---|---|
| 学数学 | 极限基础题 20 道，错题写出识别信号 |
| 提升 408 | 顺序表插入删除伪代码 + 章节题 20 道 |
| 背英语 | 高频词 30 分钟 + 阅读 1 篇定位句和错因 |
| 复习政治 | 选择题 30 道，错题归为概念/材料/记忆 |

每日任务如果超过 4 项，系统应该删减，而不是鼓励硬扛。连续低完成或出现顺延时，第二天自动降到底线日；必要时只保留顺延任务和一个核心配套任务。
