部署与备份

当前版本是 Vite 应用。开发、构建、预览都必须走 npm 脚本；直接静态打开会导致 Supabase 模块无法加载。

本机使用 一键启动 npm run start:local

自动安装依赖、寻找空闲端口、启动 Vite，并打开浏览器。

生产检查 构建预览 npm run build && npm run preview

预览的是 dist 产物，用来检查上线前的真实页面表现。

Vercel 推荐部署 Build: npm run build / Output: dist

导入 GitHub 仓库，框架选择 Vite，环境变量在 Vercel 项目设置中添加。

Supabase 云同步 执行 supabase/schema.sql

开启邮箱密码登录，配置 RLS，不在前端使用 service role key。

数据规则 双保险 云同步 + 每周 JSON

登录后云端为主，本机保留离线草稿；每周仍建议导出一次 JSON。

运行状态 本机缓存检测中 版本检测中

线上异常时先确认版本号；必要时使用账号弹窗里的清理本机缓存。
